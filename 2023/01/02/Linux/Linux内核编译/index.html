Linux内核编译

工作环境选择

VMware Workstation + Ubuntu18.04镜像

不要用Ubuntu22.04,会有问题

空间划分40GB,内存调整为8GB(不要选择默认的4GB)

下载Linux内核源代码

前往Linux Kernel下载需要的版本,本文以4.19.83版本Linux内核源代码为例,下载链接在为 https://mirrors.edge.kernel.org/pub/linux/kernel/v4.x/linux-4.19.83.tar.gz

tar xvf linux-4.19.83.tar.gz
cd linux-4.19.83

配置编译环境

sudo apt update
sudo apt-get install git fakeroot build-essential ncurses-dev xz-utils libssl-dev bc flex libelf-dev bison vim

修改配置文件

make menuconfig

打开后不做修改,save,exit,得到.config文件

vim .config

修改 CONFIG_SYSTEM_TRUSTED_KEYS 的值为 “”

vim中查找输入 /CONFIG_SYSTEM_TRUSTED_KEYS 即可

编译

make bzImage -j4

这里的4通常是CPU内核数的二倍,默认的话VMware Workstation的CPU是两核,所以这里选择4

等待一段时间之后编译成功,得到如下输出

如果最后的输出结果不同,那么检查是否内存不足,磁盘不足,运行命令不对

编译后得到vmlinux文件和bzImage文件
