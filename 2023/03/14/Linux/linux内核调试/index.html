前言

vscode + qemu 调试linux内核, 废话不多说直接开始吧

基础环境

windows + VMware + Ubuntu20.04 + Vscode

编译内核

编译linux内核的部分已经介绍过了, 见本系列的 linux内核编译

qemu

对于qemu应该不用过多介绍, 关心linux kernel的不会不知道qemu, 操作系统实验什么的也应该接触过, 简单来说就是一个非常健壮的模拟器和仿真器, 对虚拟化技术的支持很好

不建议使用apt等安装, 从源码编译的掌握力度更大. 文章编写之际qemu的最新版本为7.2.0, 也是本文采用的版本. 读者可以尝试使用当前最新的版本, 正常来说qemu的向前兼容应该是可以信任的

qemu历史版本

wget https://download.qemu.org/qemu-7.2.0.tar.xz
tar xvJf qemu-7.2.0.tar.xz
cd qemu-7.2.0

编译依赖

sudo apt-get install ninja-build
sudo apt-get install build-essential git pkg-config zlib1g-dev libglib2.0-dev libpixman-1-dev libsdl1.2-dev libspice-server-dev

这里有可能存在冲突, 解决起来有点费劲…, 或许可以换源试试, 要不就选高版本的ubuntu

这里我将qemu的安装目录放在 ~/qemu , 你可以替换为你期望的路径

./configure --prefix=~/qemu --enable-kvm  --target-list=x86_64-softmmu
make -j$(nproc)
make install

这时候 ~/qemu/bin/ 就是我们的需要的可执行文件 qemu-system-x86_64 所在的路径了

kamilu@ubuntu:~/qemu/bin$ tree .
.
├── qemu-edid
├── qemu-ga
├── qemu-img
├── qemu-io
├── qemu-nbd
├── qemu-pr-helper
├── qemu-storage-daemon
└── qemu-system-x86_64

编译busybox

wget https://busybox.net/downloads/busybox-1.36.0.tar.bz2
tar xf busybox-1.36.0.tar.bz2
cd busybox-1.36.0/
make menuconfig

配置选项中勾选 Build Static Lib

make -j$(nproc)
make install

编译得到的可执行文件保存在 _install/ 目录下, 我们可以进入这个目录的bin文件夹下使用 ll 看到一个busybox可执行文件和众多链接文件

制作initramfs

新建一个文件夹(workspace)用于后续的工作区

mkdir workspace
cd workspace

将之前的 linux kernel中的 arch/x86/boot/bzImage 和 busybox 拷贝到此目录下

此时的目录结构:

├── bzImage
│── busybox

新建一些目录, 用于文件结构, 软链接busybox

mkdir initramfs
cd initramfs
mkdir -p bin dev etc lib mnt proc sbin sys tmp var
mv ../busybox bin/
cd bin
ln -s busybox mount
ln -s busybox sh
cd ..

此时的文件结构如下所示

├── bzImage
├── initramfs
│   ├── bin
│   │   ├── busybox
│   │   ├── mount -> busybox
│   │   └── sh -> busybox
│   ├── dev
│   ├── etc
│   ├── lib
│   ├── mnt
│   ├── proc
│   ├── sbin
│   ├── sys
│   ├── tmp
│   └── var

在 initramfs/ 中新建 init 文件用于启动, 将下面的内容复制到init中

vim init
#!/bin/busybox sh
/bin/busybox --install -s /bin
mount -t devtmpfs  devtmpfs  /dev
mount -t proc      proc      /proc
mount -t sysfs     sysfs     /sys
mount -t tmpfs     tmpfs     /tmp
setsid cttyhack sh
exec /bin/sh

修改init的权限

chmod +x init

最后在根目录新建 Makefile, 复制如下内容

.PHONY: init qemu clean

init:
    cd ./initramfs && find . | cpio -ov --format=newc | gzip -9 > ../initramfs.img

qemu:
    ~/qemu/bin/qemu-system-x86_64 \
            -kernel bzImage \
            -initrd initramfs.img \
            -m 1G \
            -nographic \
            -append "earlyprintk=serial,ttyS0 console=ttyS0"

clean:
    rm initramfs.img

QEMU 启动

制作initramfs

make init

执行之后会得到 initramfs.img

使用qemu开始模拟, 注意这里的路径是 ~/qemu/bin/qemu-system-x86_64, 如果你的qemu装在了不同的地方记得修改路径

make qemu

成功启动后按下enter进入命令行, 使用ls查看目录结构

也可以在这个命令行中测试一些内容

~ # uname -r
5.4.19
~ # df
Filesystem           1K-blocks      Used Available Use% Mounted on
devtmpfs                495664         0    495664   0% /dev
tmpfs                   500336         0    500336   0% /tmp

退出使用 ctrl + A 然后按下 x 退出 QEMU 模拟

Vscode + gdb

修改Makefile, 这里使用 -s 用于gdb调试

.PHONY: init qemu clean

init:
    cd ./initramfs && find . | cpio -ov --format=newc | gzip -9 > ../initramfs.img

qemu:
    ~/qemu/bin/qemu-system-x86_64 \
            -kernel bzImage \
            -initrd initramfs.img \
            -m 1G \
            -nographic \
            -append "earlyprintk=serial,ttyS0 console=ttyS0 nokaslr" \
            -S \
            -s

clean:
    rm initramfs.img

  • 你可以在一个终端中使用 make qemu, 此时运行会卡住

  • 然后再另一个终端中使用 gdb vmlinux 调试

    进入gdb后连接1234端口

    (gdb) target remote :1234

    在start_kernel处打一个断点(此函数位于init/main.c), 然后继续

    (gdb) b start_kernel
(gdb) c

当然, 这种方法很原始, 我们也不会使用, 我们使用vscode来进行调试

生成智能补全的头文件

python3 ./scripts/gen_compile_commands.py

等待一段时间运行结束之后得到compile_commands.json

运行很慢, 这个文件很大, 5.4.19 中有44MB

使用微软的C++插件, 新建 .vscode/c_cpp_properties.json, 这里的compileCommands添加补全的路径

{
    "configurations": [
        {
            "name": "Linux",
            "includePath": [
                "${workspaceFolder}/**"
            ],
            "defines": [],
            "compilerPath": "/usr/bin/gcc",
            "cStandard": "c17",
            "cppStandard": "gnu++14",
            "intelliSenseMode": "linux-gcc-x64",
            "compileCommands": "${workspaceFolder}/compile_commands.json"
        }
    ],
    "version": 4
}

.vscode/launch.json, 设置调试的名称是 qemu-kernel-gdb, 使用本机的1234端口映射

{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "qemu-kernel-gdb",
            "type": "cppdbg",
            "request": "launch",
            "miDebuggerServerAddress": "127.0.0.1:1234",
            "program": "${workspaceFolder}/vmlinux",
            "args": [],
            "stopAtEntry": false,
            "cwd": "${workspaceFolder}",
            "environment": [],
            "externalConsole": false,
            "logging": {
                "engineLogging": false
            },
            "MIMode": "gdb"
        }
    ]
}
