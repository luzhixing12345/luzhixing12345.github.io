实验知识

从程序员的角度,我们可以认为进程总是处于下面三种状态之一.

  • 运行
  • 停止: 当收到SIGSTOP SIGTSTP SIGTTIN SIGTTOU信号进程停止,并且保持停止指导他受到一个SIGCONT信号
  • 终止: 1)收到一个信号,该信号的默认行为是终止进程 2)从主程序返回 3)调用exit函数
#include <sys/types.h>
#include <unistd.h>

pid_t fork(void);
  1. 调用一次,返回两次;父进程中返回子进程的pid,子进程中返回0
  2. 并发执行
  3. 相同但是独立的地址空间,对变量的改变是独立的
  4. 共享文件
#include <sys/types.h>
#include <sys/wait.h>

pid_t waitpid(pid_t pid, int *statusp, int options)

默认情况下options=0,waitpid挂起调用进程的执行,指导等待集合中的一个子进程返回

  • pid>0等待集合是一个单独的子进程,进程ID为pid
  • pid=-1,等待的集合是父进程所有的子进程组成的
