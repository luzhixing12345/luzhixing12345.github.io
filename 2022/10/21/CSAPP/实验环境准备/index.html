实验环境

实验要求的环境是linux系统,所以这里选择使用 WSL(Ubuntu)

安装 WSL

Windows可以使用WSL(Windows Subsystem for Linux)来使用Linux环境

  1. 打开控制面板->程序

  2. 启动或关闭 Windows 功能

  3. 开启 “适用于 Linux 的 Windows 子系统” 和 “虚拟机平台” 选项

    启用服务,重启电脑

  4. 键盘按下 Win + r 输入 cmd 后回车

    wsl --install -d Ubuntu

  5. 等待安装完毕之后,输入用户名 密码就可以进入Linux系统了,这个操作系统的名字是 Ubuntu

    如果关闭窗口后再想重新进入这个系统,只需要在命令行中输入 wsl 就可以直接进入

安装一个更加美观的 Terminal 终端

原生的命令行窗口并不好看,由于本实验可能需要大部分时间面对一个终端的显示,我们考虑选择使用一个更加美观漂亮的终端

在 terminal release 中下载最新版的安装包

注意区分 Win10/11版本,不要下错了

如果不清楚windows版本可以按下Win + r 输入 winver查看

下载后点击对应版本安装即可,接下来尝试再来按下 Win + r 输入cmd进入控制台吧,是不是好看多了?

熟悉命令行操作

Windows系统想必已经比较熟悉了,界面窗口(windows)做的很优秀,我们可以通过鼠标完成大部分的交互操作

但是Linux系统比较特殊,它主要依赖于命令行输入命令执行相关操作,我们可以使用一串串指令完成几乎所有任务.这时候我们可以抛弃鼠标,只使用键盘.

下面介绍一些常用的基本命令

  1. clear 清空所有的终端输出

    ctrl + L

  2. ls 列出目录的内容及其内容属性信息

    ls -a 列出所有文件,包含一些隐藏文件

  3. cd 切换目录

    • cd 回到根目录
    • cd … 回到上一级目录
    • cd {目录名} 进入当前路径下的文件夹

  4. mv 移动文件

    mv ../a.txt b.txt

    这条命令是指将上一级目录的 a.txt 文件移动到当前目录并且改名为 b.txt

  5. cp 复制文件

    用法同mv但是cp保留原文件mv将原文件移动走了

  6. tar 解压文件

    相关链接

Vim编辑器入门

  1. 使用vim打开一个文件(文件名为test.txt)

    vim test.txt

    如果显示vim不存在则需要先安装vim:

    sudo apt install vim

  2. 进入vim之后可以使用 ctrl + f 快速下翻页, ctrl + b 上翻页

  3. 按下键盘的 i 进入插入模式(左下角会出现-- INSERT --的字样),表示当前状态是可以编辑的,在这个状态下修改你的文件

  4. 修改完成后,按下 esc 左下角-- INSERT --消失,接着输入 :wq

    w表示保存 q表示退出

    输入之后按下回车文件修改完毕保存成功,回到之前的界面

WSL 与 Windows文件互传

WSL正常情况下会出现在文件资源管理器,可以直接点击 Linux/Ubuntu 文件夹并且在 home 目录下找到用户名

如果安装之后没有找到, 可以通过其他方式

  • win + r 输入 \\wsl$
  • 进入wsl的操作系统中,在终端中输入 explorer.exe .

我们可以直接打开两个文件夹,直接通过复制粘贴实现文件传输

除此之蛙wsl将windows中的文件挂载到 /mnt 虚拟磁盘下,默认的登录路径也是这里,我们可以直接通过mnt切换盘符及路径,比如:

kamilu@LZX:/mnt/c/Users/luzhi$ cd /mnt/d
kamilu@LZX:/mnt/d$

再利用 cp 或者 mv 命令就可以实现文件的传输了,笔者这里还是推荐第一种,比较直接

tar xvf datalab-handout.tar
