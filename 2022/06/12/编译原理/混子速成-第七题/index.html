C语言编译器行为和错误分析

这一题题目难度比较大,比较考验对于C语言的理解,以及对计算机组成原理的理解.其中指针/地址,栈/寄存器都是经常考察的地方.这道题没有什么固定的套路,只能说是见招拆招.

2008级

void foo() {
    int i;
    int a[] = {10,20};
    for (i = 0; i<=2; i++) a[i]-=3;
    return;
}

测试发现for语句处出现了死循环,分析为什么会出现死循环?

首先可以很容易观察到数组a在for循环中 i=2 一定是越界了,那么这个越界的影响就是导致了i的值被改变了所以判断 i<=2始终为真

回忆一下计组中的知识,临时变量定义在栈中,栈地址由高到低
