服务器操作系统环境搭建

本文最后更新于：6 天前

基本环境的安装

一般来说购买服务器的环节就有了安装操作系统的环节,你可以选择不同的操作系统.Centos,ubuntu等等其他unix系统

我个人是习惯使用debian操作系统的,比较稳定,网上的文档也很多.遇到的问题基本可以解决.

不同的操作系统确实有所差异,如果你遇到了什么问题不要忘记加上你的linux操作系统名字,或许才能搜索到正确的答案

选择一个操作系统之后稍事等待就帮你一键安装成功了,比较方便.

以下所有命令均在Debian操作系统中适用,Debian是我个人比较喜欢使用在服务器上的一个操作系统

每一个程序都有他们自身的用法,这里必然不可能展开讲解,如果你有兴趣的话可以查看对应的教程,这里仅仅提供了安装的方式.更加具体的配置方法和使用方法还是要各位自行摸索.

基本命令

sudo

apt-get install sudo

更新

sudo apt update

一些安装项

sudo apt install openssh-server
sudo apt install curl

SSH

这个是默认安装的

重启服务

service sshd restart

配置信息在/etc/ssh/sshd_config

记录在/var/log/auth.log

基本信息在/root/.ssh/

ufw防火墙

apt-get install ufw

软件安装

git

sudo apt install git

检查

git --version

git的SSH配置

Anaconda

visit https://www.anaconda.com/products/distribution#linux and download the lastest,最下面

通过FTP/SCP发送到服务器

bash Anaconda(TAB补全)

安装过后conda init,这个默认的不好使,需要手动将这个路径加入到环境变量中

通常来说anaconda的环境在linux中安装的位置默认是root/anaconda3,如果你更改了这个默认路径那就把下面的换成你的路径

vim ~/.bashrc
export PATH=root/anaconda3/bin:$PATH
source ~/.bashrc

npm

sudo apt install nodejs npm
nodejs --version

nginx

sudo apt install nginx

启动Nginx服务

systemctl start nginx

重启

systemctl reload nginx

停止服务

systemctl stop nginx

查看配置信息

nginx -t

纯python(应该不会有人真用纯python吧)

sudo apt install build-essential zlib1g-dev libncurses5-dev libgdbm-dev libnss3-dev libssl-dev libsqlite3-dev libreadline-dev libffi-dev curl libbz2-dev
wget https://www.python.org/ftp/python/3.9.1/Python-3.9.1.tgz
tar -xf Python-3.9.1.tgz
cd Python-3.9.1
./configure --enable-optimizations
make -j 12

为了更快完成构建,请修改-j后面的数字为你的处理器的核心数.可以通过键入nproc来找到cpu核心数。当构建过程完成时,安装Python二进制文件

这里有个小梗就是 make -j 8

很多人第一次见的时候觉得很有趣,其实8是处理器核心数

sudo make altinstall
python3.9 --version

满屏的编译输出闪过,漫长的等待之后就安装完成了

先别高兴的太早,你现在的python环境可以干干净净的,顺手把其他的环境比如flask和pytorch也装了吧,或者搞一个anaconda?写一个requirements.txt,都可以,总之先把基础的环境配好

我相信深度学习部署的应该配个环境不需要多说什么吧
